# 数据主权白皮书

## 一、核心原则

智子X 从第一天起就做了一个决定：**服务器不保存用户数据，不经手 API Key，不代理平台请求。**

```text
你的浏览器（本地）                    外部服务
┌─────────────────────┐
│  智子X网站           │
│  ├─ 采集  / AI写作   │──── 浏览器扩展 ────→ AI API（你的 Key 直连）
│  ├─ 风格模型         │──── 浏览器扩展 ────→ 平台 API（你的账号直连）

│  存储位置：          │         智子X 服务器
│  localStorage       │        ┌──────────────┐

                               └──────────────┘
```

**关键点：所有箭头都是从你的浏览器直接出去的，不经过 智子X 服务器。**

## 三、为什么用"哑管道"直连 AI

你可能听说过"哑管道"这个词——听起来很技术，但其实很简单。

### 什么是哑管道？

在智子X里，"哑管道"指的是：**你的 API Key 从你的浏览器直接发给 AI 服务商（Claude、GPT、DeepSeek 等），中间不经过任何服务器。**

浏览器扩展就像一个"透明的管道"——它只负责把你的请求原样转发出去，把 AI 的回复原样转回来，自己不看、不记、不改。

### 为什么不能用服务器代理？

很多 AI 平台会让你的请求先经过他们的服务器，再转发给 AI。这看起来方便，但有个致命问题：

**Claude 对 IP 地址非常敏感。**

如果 100 个人都通过同一个服务器 IP 去调用 Claude API，Claude 会认为这是"异常流量"，直接封禁这个 IP——结果就是：**所有人的账号都被连坐封禁。**

这不是理论风险，是真实发生过的事。

### 用哑管道的好处

1. **用你自己的网络**：你的 API Key 从你家/公司的网络直接连 AI，IP 地址是你自己的，不会和别人"共享 IP"
2. **不会被连坐**：别人的账号出问题，不会影响你
3. **隐私保护**：你和 AI 的对话内容不经过任何第三方服务器
4. **速度更快**：少了一层转发，响应更快

### 数据流对比

**传统平台（服务器代理）：**
```text
你的浏览器 ←→ 平台服务器（记录你的 Key 和请求）←→ AI API
                ↑ 风险点：多人共享 IP，容易被封
                ↑ 数据被服务器记录
```

**智子X（哑管道直连）：**
```text
你的浏览器 ←→ 浏览器扩展（透明转发）←→ AI API
              ↑ 你自己的网络，你自己的 IP
              ↑ 数据本地储存
```

**简单说：哑管道就是让你用自己的网络、自己的身份去调用 AI，不经过任何中间商。**

## 五、关于"访问所有网站"权限的说明

首次使用 AI 写作功能时，浏览器会弹窗请求"访问所有网站的数据"权限。这可能让你感到疑惑，我们来解释一下。

### 为什么需要这个权限？

因为你可能使用各种不同的 AI API：
- 官方 API：OpenAI、Claude、Gemini、DeepSeek 等
- 第三方代理：自建代理或中转服务
- 企业内网 API：公司内部部署的 AI 服务

我们无法预知你会使用哪个 API 域名，所以需要通配符权限。

### 这个权限会被滥用吗？

**不会。** 扩展只在你主动使用 AI 写作功能时，才向你配置的 API 地址发送请求。它不会：
- ❌ 访问你浏览的其他网站
- ❌ 读取你的浏览历史
- ❌ 监控你的网络活动

### 这是"可选"权限

- 安装扩展时不会自动获得此权限
- 只有首次使用 AI 功能时才请求
- 你可以拒绝，其他功能（媒体解析、公众号采集）完全正常
- 你可以随时在扩展设置中撤销

### 仍然是"哑管道"

即使有了这个权限，扩展仍然只是透明转发：
```text
你的浏览器 → 扩展（转发）→ 你配置的 AI API
```
你的 API Key 和对话内容不经过智子X服务器。

## 六、本地存储的好处和注意事项

**好处：**
1. **隐私**：你的文章、风格数据、API Key 不经过任何第三方服务器
2. **自由**：随时导出，不受平台限制
3. **免费**：本地存储不花钱
4. **速度**：数据就在你电脑上，读取毫秒级

**需要注意：** 浏览器重装或清理缓存时，数据会丢失。建议定期备份。
